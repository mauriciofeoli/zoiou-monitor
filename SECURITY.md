# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| Latest (main branch) | ✅ |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability, send an e-mail to **feoli.maa@gmail.com** with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

You will receive a response within **48 hours** confirming receipt, and a follow-up within **7 days** with a plan to address the issue.

We ask that you:
- Give us reasonable time to fix the issue before public disclosure
- Avoid accessing or modifying data belonging to other users
- Act in good faith

## Scope

In scope:
- Authentication and authorization bypass
- SQL injection / RLS bypass
- Remote code execution
- Sensitive data exposure (user emails, Telegram IDs, price data)

Out of scope:
- Denial of service attacks
- Rate limiting bypass without demonstrated real impact
- Issues requiring physical access to a device
